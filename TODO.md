# Mood Playlist Generator - Development Roadmap

## Project Overview
A Gemini-powered Spotify playlist generator that takes user mood input and generates a 15-song playlist based on that mood using Google's Gemini API and Spotify's search/playlist creation capabilities.

---

## Phase 1: Backend Foundation & Spotify Integration

### Task 1.1: Implement Spotify API Service Layer
- **Description:** Build out the empty `spotifyServices.js` file with methods to:
  - Search for songs by title and artist name
  - Get audio features and popularity data
  - Create playlists on authenticated user's account
  - Add tracks to playlists
- **Acceptance Criteria:**
  - Functions for `searchTrack()`, `createPlaylist()`, `addTracksToPlaylist()`
  - Proper error handling and validation
  - Uses Spotify Web API (credentials stored in .env)
- **Files to modify:** `server/services/spotifyServices.js`
- **Dependencies:** Spotify API credentials, `node-spotify-api` or native axios/fetch

### Task 1.2: Setup Spotify OAuth Authentication Flow (Backend)
- **Description:** Implement Spotify OAuth 2.0 authorization code flow on backend:
  - Create `/auth/spotify` endpoint to redirect to Spotify auth
  - Create `/auth/callback` endpoint to handle OAuth callback
  - Store user access tokens securely (in database or session)
  - Refresh tokens when expired
- **Acceptance Criteria:**
  - Users can authenticate with Spotify
  - Access tokens are stored and used for subsequent API calls
  - Refresh token logic prevents auth failures
- **Files to create/modify:** `server/routes/auth.js` (new), `server/app.js`
- **Dependencies:** Spotify OAuth credentials (Client ID, Client Secret)

### Task 1.3: Create Backend API Endpoint for Playlist Generation
- **Description:** Build the main `/api/playlists/generate` endpoint that:
  - Accepts user mood input
  - Calls Gemini API to get song recommendations
  - Searches Spotify for each recommended song
  - Returns matched tracks with Spotify data
- **Acceptance Criteria:**
  - Endpoint returns array of 15 songs with Spotify IDs and metadata
  - Handles cases where songs aren't found on Spotify
  - Returns proper HTTP status codes and error messages
- **Files to create/modify:** `server/routes/playlists.js` (new), `server/app.js`
- **Dependencies:** Task 1.1, working Gemini API

### Task 1.4: Create Backend API Endpoint to Save Playlist to Spotify
- **Description:** Build `/api/playlists/save` endpoint that:
  - Takes user ID and playlist name
  - Creates a new playlist on user's Spotify account
  - Adds the 15 generated tracks to the playlist
  - Returns playlist URL/ID
- **Acceptance Criteria:**
  - Playlist created on Spotify with correct name
  - All 15 tracks added successfully
  - Returns shareable Spotify URL
- **Files to create/modify:** `server/routes/playlists.js`
- **Dependencies:** Task 1.1, Task 1.2 (for user auth)

## Phase 2: Database & User Management

### Task 2.1: Design and Implement PostgreSQL Database Schema
- **Description:** Setup database schema with tables for:
  - `users` (id, spotify_id, email, access_token, refresh_token, created_at)
  - `playlists` (id, user_id, playlist_name, spotify_playlist_id, mood, songs, created_at)
  - `saved_playlists` (id, user_id, playlist_id) - for favorites
- **Acceptance Criteria:**
  - Database tables created with proper constraints and indexes
  - Foreign key relationships established
  - Migration scripts available
- **Files to create/modify:** `server/services/db/postgres.js`, migration scripts
- **Dependencies:** PostgreSQL installed and configured

### Task 2.2: Implement Database Connection & Query Functions
- **Description:** Build out `postgres.js` with helper functions:
  - `createUser()`, `getUserById()`, `updateUserTokens()`
  - `savePlaylist()`, `getUserPlaylists()`, `deletePlaylist()`
  - Proper error handling and logging
- **Acceptance Criteria:**
  - All CRUD operations working
  - Connection pooling implemented
  - SQL injection prevented (parameterized queries)
- **Files to modify:** `server/services/db/postgres.js`
- **Dependencies:** Task 2.1, `pg` library

### Task 2.3: Create Backend Routes for User Playlist History
- **Description:** Build endpoints for:
  - `GET /api/users/playlists` - get all playlists generated by user
  - `DELETE /api/playlists/:id` - delete a saved playlist
  - `GET /api/playlists/:id` - get playlist details
- **Acceptance Criteria:**
  - Users can view their playlist history
  - Users can delete old playlists
  - Proper authorization (users can only see their own playlists)
- **Files to create/modify:** `server/routes/playlists.js`
- **Dependencies:** Task 2.1, Task 2.2, Task 1.2 (auth)
